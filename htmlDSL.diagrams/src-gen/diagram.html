Basic steps on how to use xtml:

1: Define a definition with name, such as 'Container con', or 'Text introParagraph'

2: Next you can specify what your element will contain:

  2.1 A text element can contain text via e.g. Text introParagraph "Hello and welcome to my website!"
  2.2 A container can contain other elements, such as the introParagraph: "con add(introParagraph)"
  2.3 Image elements can contain a link to an image: Image exampleImage. exampleImage href("hamster.jpg")

All of these can be customized with CSS! Either directly via the elementName style("") command, or via several custom styling commands, such as grid(), flex(), font(), size() and color().

And if you want examples, you can look at the source for this html file!

Things we could've added in the future

Buttons was one thing we overlooked. While a-tags can do more or less the same, buttons are handy as a baseline.

Table support. This is very clunky in html due to the sheer number of tags needed to place things correctly. In an updated xtml, this would become a priority.

Other things we could've added include radio buttons, select-menues and sliders. These could have enhanced the flexibility of xtml by quite a bit, and allow it to mimic more of html
